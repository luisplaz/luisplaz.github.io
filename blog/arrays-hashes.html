Arryays VS Hashes

When to use each one and what they are

October 24th 2015

This week we will be talking about hashes and arrays and what they should be used for and what makes them different. Let’s start off with arrays.

ARRAYS

Arrays are ordered collections or in a simpler way a list of slots that can store a number of objects, so we can have an array of strings, of integers even an array of arrays. Also, each slot has a number assigned to it by default and in the case of arrays these numbers start from 0 onward - So the first slot in an array is slot 0 - .

Arrays as we can see are used to store objects in an ordered manner so we can access them later using their slot number. Lets review how to create arrays and extract or store objects in them.

In ruby we can create an array a number of ways. Here are 2 of them: 

a.  new_Array = Array.new(5,"def_values")
    


    b.  new_Array_Two = [ ]

Method (a) makes and array through the Array.new method, this designates an empty or filled array to a variable of our choosing in this case new_Array. The arguments inside the parenthesis are optional, these indicate first the array size – number of slots – and second the default value with which each slot starts. In this case we will have a array, new_Array, that has 5 slots with the string def_value in each one. 

    Array.new = [“def_value”, “def_value”, “def_value”, “def_value”, “def_value”]

Method (b) makes and array through the literal array constructor [ ], this designates an empty or filled array to a variable of our choosing in this case new_Array_Two. Initially if we only write the code as stated - new_Array_Two = [ ] – we create an empty array. If we want to create it with things inside from the start we can just add them into the brackets . 

 
    new_Array_Two = [ 1,2,3,4,5]

In this case, we create an array of 5 slots with numbers one – five inside each respective slot.

Both methods work in different ways but ultimately produce the same result.

Now, to add, remove or retrieve from arrays we can use the following code, for these examples we will use a 5 slot array called 

    five_Slot_Array = [1,2,3,4,5]

Add – If we desire to add more objects into our five_Slot_Array we simply add a new object in the next slot or we can replace an object for another in an already occupied slot. Example: 

    five_Slot_Array [6] = 6

So our array would now look like this: 

    five_Slot_Array = [1,2,3,4,5,6]

Retrieve – We can retrieve in a similar way to adding. We now know we have 6 slots and 6 objects so if we want to know what is in one of the slots lets say – slot 2 - we write: 

    p five_Slot_Array [2]

This should return 3 – Remember slot 0 is 1, and slot 1 is 2……

Lastly, we will explain removing.

Remove – to remove slot 3 from five_Slot_Array we simply write: 

    five_Slot_Array.delete(3)

This would result in this array 

    five_Slot_Array = [1,2,4,5]

HASHES

Now hashes are basically the same thing as array except they store objects in pair, there is a key that is a reference to the slot where the object is located and a value that point to the object inside said slot. The interesting part about hashes is that instead of having a default key like arrays do – array keys are numbers 0…onward -, hashes can have any object as a key.

So lest say we assign the key – One – to the value 1. When we want to access the value 1 we simply look up the key One in the hash. One thing that must be taken into account is that keys are unique and cannot be repeated, so there can only be a key/value pair per key.

In ruby we can create an array a number of ways. Here are 2 of them: 

  a.  new_Hash = Hash.new

  b.  new_Hash_Two = { }

Method (a) creates an empty hash that can later be populated and method (b) creates an empty hash as well but can be populated if we wanted to in this same code. Example. 

  new_Hash_Two = { 1=>2, 3=>4, 5=>6}

We use the => symbol to assing a value to a key so it works like this (key => value).

If we want to insert a new key/value into the hash we just type: 

  new_Hash_Two["Three"] = 3

This adds a key/value (Three/3) into our hash. If we want to retrieve something from the hash we use the key assigned to the value. Like this: 

  p new_Hash_Two["Three"]

Which would return 3.

Lastly if we want to delete something from the hash we write the following code: 

  new_Hash_Two.delete(“Three)

This would delete key/value associated with key “Three”

Why use one over the other

So in conclusion an array serves as a kind of list that is not very specific as to the location of its list items. The hashes are a lot more specific and take less time in searching for values as they have specific keys associated with them that can be configured to be more descriptive as to the value in the slot.
